# Seat Generation Module — Flask + MariaDB

Part of the **Bus Reservation Platform** project.  
This module is responsible for **generating numbered seats** in the database for each bus, and exposing them to other teams via a JSON API and server-rendered HTML pages.

---

## Scope

This module sits between two other teams:

```
[Bus Management team] ──► [THIS MODULE] ──► [Display / Booking team]
  creates buses               generates         reads seats &
  in the DB                   seats 1..N        their status
```

| Action | How |
|---|---|
| Generate seats for a bus | `POST /api/places/generer` |
| Read seats for a bus | `GET /api/places/<bus_id>` |
| Read seats for a specific trip | `GET /api/places/<bus_id>?trajet_id=X` |
| Delete seats for a bus | `DELETE /api/places/<bus_id>` |
| Dashboard (HTML) | `GET /` |
| Bus detail page (HTML) | `GET /bus/<bus_id>` |

---

## Database

This module uses the shared `transport_reservation` database.  
The relevant tables are:

| Table | Role |
|---|---|
| `bus` | Source of truth for capacity (15, 32, or 60) |
| `places` | Seats generated by this module |
| `reservations` | Read-only — used to display seat status |
| `agences` | Read-only — displayed in the UI |

> Run `schema.sql` once on your MariaDB instance before starting the app.  
> The `places` table is defined there with a foreign key to `bus`.

## Installation

### Prerequisites

- Python 3.10+
- MariaDB or MySQL running locally
- The `transport_reservation` database created (see `schema.sql`)

### Steps

```bash
# 1. Clone the repository and switch to your branch
git clone <repo-url>
cd projet_places
git checkout feature/gestion-places

# 2. Create a virtual environment
python -m venv venv
source venv/bin/activate        # Linux / macOS
venv\Scripts\activate           # Windows

# 3. Install dependencies
pip install -r requirements.txt

# 4. Initialize the database
mysql -u root -p < schema.sql
```

## Configuration

Edit the `app.config.from_mapping(...)` block in `app/__init__.py`:

```python
app.config.from_mapping(
    DB_HOST     = "localhost",
    DB_PORT     = 3306,
    DB_USER     = "root",
    DB_PASSWORD = "your_password",
    DB_NAME     = "transport_reservation",
)
```

## API Reference

### `POST /api/places/generer`

Generates seats for a bus. Accepted capacities: **15, 32, 60**.

**Request body:**
```json
{ "bus_id": 3, "capacite": 15 }
```
`capacite` is optional — if omitted, the value stored in the `bus` table is used.

**Responses:**

| Status | Meaning |
|---|---|
| `201` | Seats successfully generated |
| `400` | Missing or invalid fields |
| `409` | Seats already exist for this bus |
| `422` | Capacity not in (15, 32, 60) |

**201 example:**
```json
{
  "bus_id": 3,
  "immatriculation": "MR-001-AB",
  "capacite": 15,
  "places_generees": 15,
  "message": "15 places générées avec succès pour le bus 3."
}
```

---

### `GET /api/places/<bus_id>`

Returns all seats for a bus with their reservation status.

Optional query param: `?trajet_id=X` — filters status for a specific trip.

**200 example:**
```json
{
  "bus_id": 3,
  "trajet_id": null,
  "total": 15,
  "places": [
    { "id_place": 1, "numero_place": 1, "bus_id": 3, "statut_place": "libre" },
    { "id_place": 2, "numero_place": 2, "bus_id": 3, "statut_place": "confirmee" }
  ]
}
```

Seat statuses: `libre` / `en_attente` / `confirmee`

---

### `DELETE /api/places/<bus_id>`

Deletes all seats for a bus. Use before re-generating with a different capacity.

**200 example:**
```json
{ "bus_id": 3, "places_supprimees": 15, "message": "15 place(s) supprimée(s) pour le bus 3." }
```

---

### `GET /api/places/bus`

Returns all buses with their generation status.

## Integration Notes for Other Teams

### For the Bus Management team
Call `POST /api/places/generer` with `{ "bus_id": <id> }` right after inserting a bus.  
The capacity will be read automatically from the `bus` table — no need to pass it.

### For the Display / Booking team
Call `GET /api/places/<bus_id>?trajet_id=<id>` to get seats with their status for a given trip.  
The response is ready to render — each seat has a `statut_place` field (`libre`, `en_attente`, `confirmee`).
